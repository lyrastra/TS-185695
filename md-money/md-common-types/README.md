# md-common-types

**Владелец/команда:** Команда Архитектуры

## Назначение

Библиотека типовых идентификаторов (ID-обёрток) для использования в различных корпоративных доменах. Обеспечивает строгую типизацию идентификаторов пользователей, организаций, ролей, тарифов и других сущностей для повышения надёжности и читаемости кода.

## Основные возможности
- Строго типизированные структуры для идентификаторов (UserId, FirmId, RoleId, TariffId)
- Безопасные преобразования между int и ID-типами
- Сравнение, хеширование, поддержка значений "Unidentified"
- Унификация работы с идентификаторами в разных слоях приложения

## Быстрый старт
1. Подключите сабмодуль:
   ```sh
   git submodule add https://gitlab.mdtest.org/development/infra/md-common-types.git
   ```
2. Добавьте проект из папки `src/` в ваш solution (.sln) через Visual Studio или вручную.
3. Используйте типы в своём коде:
   ```csharp
   using Moedelo.Common.Types;
   UserId userId = new UserId(123);
   int id = (int)userId;
   if (userId == UserId.Unidentified) { /* ... */ }
   ```

## Зависимости
- Не зависит от других сабмодулей
- Требуется .NET Standard 2.0 или выше

## Совместимость и ограничения
- Совместим с .NET Standard 2.0+, .NET Core 2.0+, .NET 5/6/7/8+
- Не содержит бизнес-логики, только типовые структуры
- Не рекомендуется использовать для хранения бизнес-логики или сложных операций

## Состав и проекты
- Moedelo.Common.Types (Moedelo.Common.Types.csproj): типовые структуры-идентификаторы (UserId, FirmId, RoleId, TariffId), TargetFramework: netstandard2.0

## Best practices
- Используйте типовые ID-структуры вместо int/long для идентификаторов в публичных API и слоях бизнес-логики
- Не смешивайте разные ID-типы между собой
- Для неинициализированных значений используйте Unidentified
